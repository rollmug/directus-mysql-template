If using PM2 to automatically start processes:

1. Update the docker-compose.yaml file so that all services use "restart: on-failure" (or even better: unless-stopped)
2. Stop all running services (npm run stop)

3. Install and configure:

npm install -g pm2
pm2 --version

pm2 start ecosystem.config.cjs

pm2 startup (then paste the command it outputs)

pm2 save

4. Now, all services to start up properly on reboot or power failure.

Other tips for Windows/WSL 2:

1. Enable Systemd in WSL 

Open your WSL terminal and edit the configuration file: sudo nano /etc/wsl.conf

Add the following lines:

[boot]
systemd=true

2. Auto-Start Docker

Once systemd is enabled, you can set Docker to start whenever the WSL instance is triggered:

If using Docker Desktop: Ensure "Start Docker Desktop when you log in" is checked in the Docker Desktop settings.

3. Test the setup:

On reboot, open a terminal and run the following commands individually: 

systemctl status pm2-<username>
pm2 status
docker compose ps

Make sure everything has launched properly, enabled, etc.
